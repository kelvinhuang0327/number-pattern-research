# 🤖 智能自動學習系統使用指南

## 📋 功能概述

智能自動學習系統是一個**自我學習的 AI 系統**，可以：
1. ✅ **自動評估所有預測策略** - 找出最佳方法
2. ✅ **優化策略參數** - 使用遺傳算法
3. ✅ **持續學習改進** - 定期自動優化

---

## 🎯 核心功能

### 1️⃣ 智能策略評估

**功能**: 自動測試所有可用的預測策略，找出在當前數據集上表現最好的策略。

**可評估的策略**:
- 🔢 **核心統計策略**: 頻率分析、貝葉斯統計、馬可夫鏈、蒙地卡羅模擬
- 🎲 **民間策略**: 奇偶平衡、區域平衡、冷熱混合
- 🚀 **高級策略**: 隨機森林/KNN相似度、集成預測

**評估指標**:
- **成功率**: 中 3 個以上的期數比例
- **平均命中數**: 平均每期中幾個號碼
- **命中率分佈**: 中 0/1/2/3/4/5/6 個的分佈
- **vs 理論值**: 實際成功率 vs 隨機猜測的理論概率 (1.765%)
- **綜合評分**: 加權計算的總體評分 (0-100分)

**使用方法**:
```
1. 確保已同步數據到後端 (點擊「同步數據到後端」)
2. 選擇測試集比例 (推薦 20%)
3. 點擊「開始評估策略」
4. 等待評估完成 (約 30-60 秒)
5. 查看最佳策略和所有策略的評估結果
```

**評估結果說明**:

### 2️⃣ 參數自動優化

**功能**: 使用遺傳算法優化單個策略的參數（權重、係數等）。

**使用方法**:
```
1. 確保已同步數據到後端
2. 設置遺傳代數 (5-100，推薦 20)
3. 設置種群大小 (10-100，推薦 30)
4. 點擊「開始優化」
5. 等待優化完成 (約 1-3 分鐘)
6. 查看最佳配置和進化趨勢圖
```

**參數說明**:
- **遺傳代數**: 進化的迭代次數，越大越精確但耗時更長
- **種群大小**: 每代的個體數量，越大越容易找到最優解但耗時更長

---

### 3️⃣ 自動排程優化

**功能**: 每天自動執行策略評估和參數優化，持續學習改進。

**使用方法**:
```
1. 設置每日執行時間 (推薦凌晨 2:00)
2. 點擊「啟動排程」
3. 系統會每天自動執行優化
4. 可以隨時點擊「停止排程」取消
```

## 🚀 完整工作流程

### 初次使用

```
步驟 1: 上傳 CSV 數據
  ↓
步驟 2: 同步數據到後端
  ├─ 點擊「同步數據到後端」
  └─ 等待同步完成
  ↓
步驟 3: 評估所有策略
  ├─ 選擇測試集比例 (20%)
  ├─ 點擊「開始評估策略」
  └─ 查看最佳策略
  ↓
步驟 4: (可選) 優化最佳策略的參數
  ├─ 設置遺傳代數和種群大小
  ├─ 點擊「開始優化」
  └─ 查看優化結果
  ↓
步驟 5: (可選) 啟動自動排程
  ├─ 設置執行時間
  └─ 點擊「啟動排程」
```

### 日常使用

```
每天:
  - 系統自動優化 (如果啟用排程)

每週:
  - 手動點擊「開始評估策略」檢查是否有新的最佳策略

每月:
  - 上傳新的 CSV 數據
  - 重新同步到後端
  - 重新評估策略
```

## 💡 最佳實踐

### 評估策略時

1. ✅ **確保數據充足** - 至少 50 期數據
2. ✅ **定期重新評估** - 每週或有新數據時
3. ✅ **關注成功率和 vs 理論值** - 不只看評分
4. ✅ **比較多個策略** - 查看所有策略的結果

### 參數優化時

1. ✅ **先評估策略** - 確定最佳策略後再優化
2. ✅ **適當設置代數** - 20 代通常足夠
3. ✅ **不要過度優化** - 避免過擬合
4. ✅ **定期重新優化** - 每月或有新數據時

### 排程使用時

1. ✅ **選擇低峰時段** - 凌晨 2:00
2. ✅ **定期檢查狀態** - 確保排程正常運行
3. ✅ **保持數據同步** - 確保後端數據是最新的

## 🐛 常見問題

### Q: 為什麼評估失敗？
**A**: 檢查以下項目：
1. 是否已同步數據到後端
2. 數據量是否充足 (至少 40 期)
3. 後端 API 是否正常運行
4. 查看瀏覽器控制台錯誤訊息

### Q: 評估需要多長時間？
**A**: 取決於數據量和測試集比例：
- 10% 測試集: 約 20-30 秒
- 20% 測試集: 約 30-60 秒
- 30% 測試集: 約 60-120 秒

### Q: 如何解讀評估結果？
**A**: 關鍵指標：
1. **評分 > 70**: 優秀
2. **評分 60-70**: 良好
3. **評分 < 60**: 一般
4. **vs 理論值 > 10**: 明顯優於隨機
5. **成功率 > 10%**: 可接受

### Q: 最佳策略會變嗎？
**A**: 會的！最佳策略可能因為：
- 數據量增加
- 彩票規則變化
- 歷史趨勢改變

建議每週重新評估一次。

### Q: 能保證中獎嗎？
**A**: **不能！**彩票本質是隨機的。此系統只是：
- 使用統計方法提高命中概率
- 比純隨機猜測更好
- 但無法保證中獎

請理性投注，不要沉迷！

## 🔬 技術原理

### 滾動驗證 (Rolling Validation)

```
測試原理:
  對每一期進行預測驗證
  ├─ 期數 001: 使用 [001之前的數據] → 預測 001 → 驗證
  ├─ 期數 002: 使用 [002之前的數據] → 預測 002 → 驗證
  ├─ 期數 003: 使用 [003之前的數據] → 預測 003 → 驗證
  └─ ...

這樣可以:
  ✅ 模擬真實預測環境
  ✅ 避免數據洩漏
  ✅ 準確評估策略性能
```

### 綜合評分計算

```python
score = (
    success_rate * 0.5 +          # 成功率 (權重 50%)
    (avg_hits / 6.0) * 0.3 +      # 平均命中數 (權重 30%)
    (vs_theory / 3.0) * 0.2       # vs 理論倍數 (權重 20%)
) * 100

範例:
  成功率 = 25.3% → 0.253
  平均命中數 = 2.8 → 2.8/6 = 0.467
  vs 理論 = 14.3 → 14.3/3 = 4.77 (限制為 1.0)

  score = (0.253*0.5 + 0.467*0.3 + 1.0*0.2) * 100
        = (0.127 + 0.140 + 0.200) * 100
        = 46.7 分
```
