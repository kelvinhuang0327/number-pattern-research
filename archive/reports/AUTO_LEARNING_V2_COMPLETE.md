## 🎯 核心設計理念

### 問題定義
- 後端有 **9 種預測策略** (頻率、貝葉斯、馬可夫、蒙地卡羅、集成等)
- 不同數據集上，不同策略表現不同
- 需要**自動找出**當前數據集上最佳的策略

### 解決方案
```
智能策略評估系統
    ↓
使用滾動驗證測試所有策略
    ↓
計算多維度性能指標
    ↓
綜合評分排序
    ↓
推薦最佳策略
```

## 📊 評估指標詳解

### 1. 成功率 (Success Rate)
- **定義**: 中 3 個以上的期數比例
- **計算**: 成功期數 / 總測試期數
- **權重**: 50%
- **示例**: 38/150 = 25.3%

### 2. 平均命中數 (Average Hits)
- **定義**: 平均每期中幾個號碼
- **計算**: 總命中數 / 總測試期數
- **權重**: 30%
- **示例**: 420/150 = 2.8 個

### 3. vs 理論值 (vs Theory)
- **定義**: 實際成功率 vs 隨機猜測理論概率的倍數
- **理論值**: 大樂透中3個 = 1.765%
- **計算**: 實際成功率 / 理論成功率
- **權重**: 20%
- **示例**: 25.3% / 1.765% = 14.3倍

### 4. 綜合評分 (Score)
```
score = (
    success_rate * 0.5 +
    (avg_hits / 6.0) * 0.3 +
    min(vs_theory / 3.0, 1.0) * 0.2
) * 100

示例:
  成功率 = 25.3% → 0.253
  平均命中 = 2.8 → 2.8/6 = 0.467
  vs理論 = 14.3 → min(14.3/3, 1.0) = 1.0

  score = (0.253*0.5 + 0.467*0.3 + 1.0*0.2) * 100
        = (0.127 + 0.140 + 0.200) * 100
        = 46.7 分
```

## 🔬 滾動驗證機制

### 原理
```
假設有 200 期數據，測試比例 20%
  ↓
測試範圍: 最後 40 期 (161-200)
  ↓
期數 161: 使用 [1-160] 預測 161 → 驗證
期數 162: 使用 [1-161] 預測 162 → 驗證
期數 163: 使用 [1-162] 預測 163 → 驗證
...
期數 200: 使用 [1-199] 預測 200 → 驗證
  ↓
統計 40 期的命中率和成功率
```

### 優勢
✅ **模擬真實環境** - 每次預測都使用"該期之前"的數據
✅ **避免數據洩漏** - 不會用未來數據預測過去
✅ **準確評估性能** - 反映策略在實際使用中的表現

## 🚀 使用流程

### Step 1: 準備數據
```
1. 上傳 CSV 數據到前端
2. 點擊「同步數據到後端」
3. 確保後端有完整數據
```

### Step 2: 評估策略
```
1. 選擇彩券類型（例如: 大樂透）
2. 選擇測試集比例（推薦 20%）
3. 點擊「開始評估策略」
4. 等待 30-60 秒
```

### Step 3: 查看結果
```
1. 查看最佳策略信息
   - 策略名稱
   - 綜合評分
   - 成功率
   - 平均命中數
   - vs 理論值倍數

2. 查看所有策略排名
   - 完整性能對比表格
   - 從高到低排序
```

### Step 4: (可選) 使用最佳策略
```
可以調用 /api/auto-learning/predict-with-best
自動使用評估出的最佳策略進行預測
```
