# 彩券數據上傳工具使用說明

## 概述

本專案支援多種彩券數據上傳格式，包括 CSV 和 TXT 格式。所有工具都會自動處理重複數據，確保數據庫的完整性。

## 支援的彩券類型和格式

| 彩券類型 | 格式 | 檔案範例 |
|---------|------|----------|
| 大樂透 | CSV | 大樂透_2025.csv |
| 威力彩 | CSV | 威力彩_2025.csv |
| 今彩539 | TXT | 今彩539_2025_12.txt |

## 上傳工具

### 1. 通用上傳工具（推薦）

**檔案**: `tools/upload_lottery_data.py`

**功能**: 自動搜尋並上傳所有彩券類型的數據

**使用方法**:
```bash
python3 tools/upload_lottery_data.py
```

**特點**:
- 自動識別所有支援的彩券類型

## 數據格式說明

### CSV 格式（大樂透、威力彩）

標準台灣彩券官方 CSV 格式：

```
遊戲名稱,期別,開獎日期,銷售總額,銷售注數,總獎金,獎號1,獎號2,獎號3,獎號4,獎號5,獎號6,特別號
大樂透,113000001,2024/01/02,153681234,3073625,82950000,5,12,18,23,35,42,19
```

**欄位說明**:
- 期別: 第2欄
- 開獎日期: 第3欄（格式: YYYY/MM/DD）
- 獎號1-6: 第7-12欄
- 特別號: 第13欄

### TXT 格式（今彩539）

官方格式，每筆記錄包含 5 行：

```
第114000295期
開獎日期:114/12/06
大小順序
開出順序
0622232432
```

**格式說明**:
- 第1行: 期號（格式: 第XXXXXXXXX期）
- 第2行: 開獎日期（格式: 開獎日期:YYY/MM/DD，民國年）
- 第3行: 大小順序（忽略）
- 第4行: 開出順序（忽略）
- 第5行: 號碼（10位數字，每2位為一個號碼）

**號碼解析範例**:
- 原始: `0622232432`
- 解析: `06`, `22`, `23`, `24`, `32`
- 結果: [6, 22, 23, 24, 32]

## API 端點

### 驗證並上傳文件

**端點**: `POST /api/data/validate-csv`

**參數**:
- `file`: 上傳的檔案（CSV 或 TXT）
- `lottery_type`: 彩券類型（BIG_LOTTO, POWER_LOTTO, DAILY_539）

**支援格式**:
- `.csv` - 自動使用 CSV 解析器
- `.txt` - 自動使用 TXT 解析器

**回應範例**:
```json
{
  "valid": true,
  "errors": [],
  "parsed_data": [...],
  "stats": {
    "total_rows": 100,
    "valid_rows": 100,
    "error_rows": 0
  }
}
```

## 數據驗證規則

### 共通規則
- 日期格式必須正確
- 期號必須為數字
- 號碼不可重複
- 自動跳過重複記錄

### 大樂透
- 號碼範圍: 1-49
- 選取數量: 6 個
- 特別號範圍: 1-49

### 威力彩
- 號碼範圍: 1-38
- 選取數量: 6 個
- 特別號範圍: 1-8

### 今彩539
- 號碼範圍: 1-39
- 選取數量: 5 個
- 無特別號

## 錯誤處理

### 常見錯誤

1. **檔案編碼錯誤**
   - 確保 CSV 使用 UTF-8-BOM 編碼
   - TXT 檔案使用 UTF-8 編碼

2. **格式錯誤**
   - 檢查欄位數量是否正確
   - 確認日期格式
   - 驗證號碼範圍

3. **重複記錄**
   - 系統會自動跳過重複的期號
   - 不會覆蓋已存在的數據

## 資料庫位置

資料庫檔案: `lottery-api/data/lottery.db`

## 驗證上傳結果

使用 SQLite 命令查詢:

```bash
# 查看所有數據統計
sqlite3 lottery-api/data/lottery.db "SELECT lottery_type, COUNT(*) FROM draws GROUP BY lottery_type"

# 查看今彩539最新數據
sqlite3 lottery-api/data/lottery.db "SELECT draw, date, numbers FROM draws WHERE lottery_type = 'DAILY_539' ORDER BY draw DESC LIMIT 10"

# 查看大樂透最新數據
sqlite3 lottery-api/data/lottery.db "SELECT draw, date, numbers, special FROM draws WHERE lottery_type = 'BIG_LOTTO' ORDER BY draw DESC LIMIT 10"
```

## 注意事項

1. **自動轉換民國年**: TXT 格式的民國年會自動轉換為西元年
2. **號碼排序**: 上傳時會自動將號碼排序
3. **特別號處理**: 今彩539 的特別號會設為 NULL
4. **批次上傳**: 建議使用通用上傳工具一次處理所有檔案
5. **重複檢查**: 系統會根據彩券類型和期號檢查重複

## 疑難排解

### 上傳失敗
1. 檢查檔案路徑是否正確
2. 確認檔案格式符合規範
3. 查看錯誤日誌獲取詳細資訊

### 數據不正確
1. 使用驗證端點檢查數據
2. 確認號碼範圍是否正確
3. 檢查日期格式

### 找不到檔案
1. 確認檔案放在正確的目錄
2. 檢查檔案命名格式
3. 排除「加開」等特殊檔案
