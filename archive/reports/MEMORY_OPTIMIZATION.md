# 記憶體優化指南

## 問題說明

當載入大量彩券數據時（例如 68,243 筆記錄），瀏覽器可能會出現記憶體不足的警告，導致功能異常。

## 已實施的優化

### 1. 自動數據限制
- **限制**：記憶體中最多保留 30,000 筆數據
- **策略**：自動保留最新的數據，移除較舊的記錄
- **位置**：`src/core/DataProcessor.js`

```javascript
this.maxDataInMemory = 30000; // 可調整此數值
```

### 2. 記憶體監控
- **功能**：每 10 秒自動檢查記憶體使用率
- **警告**：當記憶體使用超過 80% 時會顯示通知
- **位置**：`src/utils/MemoryOptimizer.js`

### 3. 數據優化建議
系統會自動分析數據量並提供建議：
- 超過 50,000 筆：建議使用 IndexedDB
- 超過 20,000 筆：建議使用分頁載入
- 記憶體使用率 > 80%：建議清除緩存

## 使用建議

### 方案 1：分批上傳（推薦）
不要一次上傳所有 CSV 文件，而是分批上傳：

```
第一批：大樂透、威力彩、今彩539（< 1,000 筆）
第二批：3星彩、4星彩、39樂合彩（< 1,000 筆）
第三批：賓果賓果（61,712 筆）- 單獨處理
```

### 方案 2：只上傳需要的彩券類型
如果只分析特定彩券，只上傳該類型的 CSV 文件。

### 方案 3：清除不需要的數據
使用「清除所有數據」按鈕，釋放記憶體後再載入新數據。

### 方案 4：調整數據限制
如果電腦記憶體充足，可以調整限制：

在 `src/core/DataProcessor.js` 中：
```javascript
this.maxDataInMemory = 50000; // 增加至 50,000 筆
```

## 記憶體使用情況查看

打開瀏覽器控制台（F12），查看記憶體使用：
```javascript
// 手動檢查記憶體
memoryOptimizer.checkMemoryUsage()

// 查看優化建議
memoryOptimizer.getOptimizationSuggestions(dataLength)
```

## 瀏覽器建議

### Chrome/Edge（推薦）
- 記憶體管理最佳
- 支援 `performance.memory` API
- 建議分配 4GB+ RAM 給瀏覽器

### Firefox
- 記憶體管理良好
- 需要在 `about:config` 啟用 `dom.enable_performance_observer`

### Safari
- 記憶體限制較嚴格
- 建議減少數據量至 20,000 筆以下

## 常見問題

### Q1: 網頁卡住或無回應？
**解決方法**：
1. 重新整理頁面（Ctrl+Shift+R 清除緩存）
2. 清除所有數據
3. 分批上傳較少的文件

### Q2: 顯示「記憶體使用率過高」警告？
**解決方法**：
1. 清除不需要的數據
2. 關閉其他瀏覽器分頁
3. 減少上傳的數據量

### Q3: 模擬測試變慢？
**解決方法**：
1. 減少測試的年份範圍
2. 選擇較少的預測策略
3. 清除圖表數據緩存

### Q4: 如何完全避免記憶體問題？
**最佳實踐**：
1. 只上傳需要分析的彩券類型
2. 定期清除舊數據
3. 使用較新的瀏覽器版本
4. 確保電腦有充足的記憶體（建議 8GB+）

## 技術細節

### 數據結構優化
每筆數據只保留必要欄位：
- `draw`：期號
- `date`：日期
- `lotteryType`：彩券類型
- `numbers`：號碼陣列
- `special`：特別號

### 自動垃圾回收
系統會在以下時機建議垃圾回收：
- 數據清除後
- 切換彩券類型時
- 模擬測試完成後

### IndexedDB 存儲
大數據會自動存儲到 IndexedDB：
- 容量限制：~50MB+（視瀏覽器而定）
- 持久化：關閉瀏覽器後仍保留
- 性能：比 localStorage 快 10 倍以上

## 監控指標

系統會追蹤以下指標：
- `usedJSHeapSize`：已使用的 JS 堆疊大小
- `totalJSHeapSize`：總分配的堆疊大小
- `jsHeapSizeLimit`：堆疊大小上限
- `usagePercent`：使用率百分比
