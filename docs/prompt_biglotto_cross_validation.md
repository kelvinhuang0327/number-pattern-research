# Prompt: 威力彩策略移植大樂透 — 全面長短期回測驗證

> 用途：將此 Prompt 交給任何 AI 助理（Gemini / Claude / 其他），要求其針對大樂透執行標準化回測。
> 版本：v1.0 (2026-02-10)

---

## 任務目標

將以下在**威力彩 (Power Lotto, 1-38 選 6)** 上經驗證有效的策略，**逐一移植**到**大樂透 (Big Lotto, 1-49 選 6)** 上進行回測驗證。

目的是確認每個策略在大樂透的號碼空間（49 vs 38）下：
1. 是否仍具備正 Edge（優於隨機基準）
2. 是否存在長線衰減（短期有效但長期失效）
3. 是否適合作為大樂透的正式推薦策略

## 🔴 強制性回測規則（不可違反）

### 規則 1: N 注基準必須對齊

**所有 Edge 必須使用正確的 N 注隨機基準計算**，而非單注基準。

```
正確公式: P(N注至少一注 M3+) = 1 - (1 - P_single)^N

大樂透 P_single = C(6,3)*C(43,3) / C(49,6) = 1.86%

大樂透各注數隨機基準:
  1注: 1.86%
  2注: 3.69%
  3注: 5.49%
  4注: 7.25%
  7注: 12.34%
```

**❌ 錯誤示範**: 「3注命中率 4.73%，高於單注 1.86%，所以有效」
**✅ 正確示範**: 「3注命中率 4.73%，低於 3注基準 5.49%，Edge = -0.76%，比隨機差」

### 規則 2: 長短期三階段回測

每個策略必須在以下 **三個窗口** 下分別回測：

| 窗口 | 期數 | 用途 |
|------|------|------|
| 短期 | **150 期** | 快速篩選，檢測短期爆發力 |
| 中期 | **500 期** | 排除幸運窗口 |
| 長期 | **1500 期** | 確認長線穩定性，排除過擬合 |

> ⚠️ **關鍵**: 500 期仍可能是幸運窗口（已有前車之鑑：結構過濾 3注從 500期 +1.92% 崩壞至 1000期 -1.38%）。
> 長期回測是唯一可信的結論依據。

### 規則 3: 防數據洩漏協議

```python
# ✅ 正確: 預測時只能看到該期之前的數據

# ❌ 錯誤: 使用全域統計或未來數據

```

### 規則 4: 固定隨機種子

```python
import random
import numpy as np

SEED = 42  # 官方基準種子
random.seed(SEED)
np.random.seed(SEED)
```

如果策略包含隨機成分，必須額外使用 10 個種子 (42-51) 進行穩定性測試，並報告平均值 ± 標準差。

### 規則 5: 號碼空間適配

將策略從威力彩 (1-38) 移植到大樂透 (1-49) 時，必須調整：
- `max_num`: 38 → 49
- `pick_count`: 維持 6
- 區間劃分: 如果策略使用 Zone (分區)，必須重新計算區間邊界
- 頻率/週期分析: 窗口大小可能需要調整（49 號碼的期望間隔約 8.2 期 vs 38 號碼的 6.3 期）

## ⚠️ 特別警告

1. **不要用「命中率高於單注基準」來宣稱多注策略有效**。3注策略的基準是 5.49%，不是 1.86%。
2. **不要只跑 150 期就下結論**。歷史證明 500 期仍可能是幸運窗口。
3. **不要省略隨機基準欄位**。每一行結果都必須附帶對應注數的正確隨機基準。
4. **如果某策略包含主觀參數調整（如窗口大小），必須說明調整理由**，不可為了結果好看而反覆調參。
5. **Frequency Baseline 的單注命中率不應低於 1.86%**。如果低於此值，說明回測方法本身有錯誤，請先排查。
