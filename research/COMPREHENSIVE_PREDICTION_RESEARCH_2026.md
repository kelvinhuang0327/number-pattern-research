# 大樂透預測方法科學研究總報告

**報告日期**: 2026-02-23
**研究者**: 量化分析團隊
**驗證框架**: Walk-forward OOS + Permutation Test + Bonferroni 校正
**涵蓋範圍**: 大樂透 (49選6) 與威力彩 (38選6+8選1)

---

## 摘要 (Executive Summary)

本報告系統性地回顧了過去數月對台灣彩券預測方法的全部研究成果。通過嚴格的走前驗證 (Walk-forward)、排列檢定 (Permutation Test)、與 Bonferroni 多重比較校正，我們得出以下核心結論：

1. **現行已驗證策略仍為最佳**：大樂透 5注 TS3+M+FO (Edge +1.77%, z=2.40, p=0.008) 與威力彩 3注 PP3 (Edge +2.30%, p=0.015) 維持統計顯著性。
2. **所有「新發現」假說均被否決**：包括 Regime-Gated Formula (RGF)、FCF 替換方案、覆蓋率最大化信號模型，在嚴格驗證下均無法通過統計門檻。
3. **彩券預測的理論上限極低**：即使最佳策略的絕對 Edge 也僅在 +1~2% 範圍，遠不足以產生正期望值的投資回報。

## 一、遊戲結構與數學基礎

### 1.1 大樂透 (Big Lotto) 規則

| 參數 | 數值 |
|------|------|
| 號碼範圍 | 1~49 |
| 每注選號 | 6 個 |
| 總組合數 | C(49,6) = 13,983,816 |
| 單注 M3+ 概率 | P₁ = C(6,3)×C(43,3) / C(49,6) = 1.86% |
| 3注 M3+ 概率 | P₃ = 1-(1-0.0186)³ = 5.48% |
| 5注 M3+ 概率 | P₅ = 1-(1-0.0186)⁵ = 8.96% |

### 1.2 威力彩 (Power Lotto) 規則

| 參數 | 數值 |
|------|------|
| 第一區號碼 | 1~38 選 6 |
| 第二區特別號 | 1~8 選 1 |
| 單注 M3+ 概率 | P₁ = 3.87% |
| 3注 M3+ 概率 | P₃ = 1-(1-0.0387)³ = 11.17% |

### 1.3 理論不可能性邊界

根據資訊理論，均勻獨立抽球的彩券在數學上不存在可利用的預測信號。然而，實際開球機械裝置可能引入微弱的時序相關性（mechanical bias）。本研究的核心問題是：**這種微弱信號是否在統計上可測且穩定？**

## 二、方法論：三階段嚴格驗證協定

所有策略必須通過以下三階段驗證才被視為「有效」：

### 階段 1：Walk-forward OOS 驗證
- **原則**: 每一期 t 的預測僅使用 `history[:t]`（嚴格時序隔離）
- **窗口**: 1500 期 (約 12~15 年的完整樣本)
- **指標**: M3+ 命中率 vs 隨機基準

### 階段 2：Permutation Test (P3)
- **原則**: 將開獎序列隨機洗牌 (n≥200次)，破壞時序結構
- **判定**: 真實 Edge 必須顯著高於洗牌後的 Edge 分布
- **門檻**: p < 0.05 (洗牌後 ≥95% 次的 Edge 小於真實值)
- **意義**: 確認 Edge 來自時序結構而非號碼分布性質

### 階段 3：三窗口穩定性
- **窗口**: 150 / 500 / 1500 期
- **判定**: 三窗口 Edge 均為正 → STABLE (可採納)
- **拒絕**: 短期正、長期負 → SHORT_MOMENTUM (過擬合)

### 多重比較校正
- 當同時測試 N 個策略/公式時，適用 Bonferroni 校正: α' = 0.05/N
- 未通過校正的「顯著」結果視為假陽性

**策略組成**:
1. **注1 (Fourier Rhythm)**: FFT 週期分析，window=500

3. **注3 (TailBalance)**: 尾數(0-9)均衡覆蓋

5. **注5 (頻率正交)**: 排除前4注已選號碼後的頻率排序

**關鍵設計原則**:
- 5注 × 6號 = 30個號碼，零重疊 → 最大化覆蓋 30/49 = 61.2%
- 每注使用不同的統計特徵維度 → 正交性降低共線性
- Bet3 (TailBalance) 勝過 FreqOrt (見第五節 FCF 驗證)

## 四、被否決的假說一覽

### 4.1 Regime-Gated Formula (RGF) — 大樂透

**假說**: 使用 GMM 將市場狀態分為3類 (低熵穩定/高熵混沌/區間聚集)，在不同狀態下使用不同的號碼得分公式 (freq×gap, freq×markov 等乘法組合) 可獲取超額 Edge。

**原始宣稱**:
- State 1 × freq×markov: Edge +2.74%
- State 0 × freq×gap: Edge +1.05%

**Walk-forward 實測結果** (1500期):

| State | Formula | 原始宣稱 | 實測 Edge | p-value | 判定 |
|-------|---------|---------|-----------|---------|------|
| State 0 | freq×gap | +1.05% | **-9.97%** | 0.661 | 反轉 |
| State 1 | freq×markov | +2.74% | **-29.57%** | 0.892 | 崩潰 |
| State 0 | freq+markov | — | +67.21% | 0.052 | 邊際但Bonferroni失敗 |
| State 0 | freq_only | — | +41.48% | 0.160 | 不顯著 |

**Permutation Test 結果**:
- 最佳: State 0 × freq+markov, Perm p=0.053 (未通過 0.05 門檻)
- State 0 × freq_only, Perm p=0.024 (通過) → 但這只是基準頻率公式，不含 Regime 信息

**失敗原因**:
1. **GMM 數據洩漏**: 原始研究在全量數據上訓練 GMM 後再評估 → 未來洩漏
2. **乘法公式噪音放大**: freq×gap, freq×markov 等乘法組合在 Walk-forward 下全部為負 Edge
3. **小樣本**: State 1 僅 89 期 (原始) / 687 期 (Walk-forward 但效果消失)

**最終裁決**: **全面撤回 (REJECTED)**

### 4.2 Regime-Gated Formula (RGF) — 威力彩

**Walk-forward 實測結果** (1500期):

| 最佳組合 | 實測 Edge | Perm p-value |
|---------|-----------|-------------|
| State 2 × gap×markov | +19.54% | **0.466** |
| State 1 × gap×markov | +16.28% | **0.550** |
| State 0 × gap×markov | +10.47% | **0.643** |

所有 Permutation p-values ≥ 0.466 → Edge 完全在隨機噪音範圍內。

**最終裁決**: **全面撤回 (REJECTED)**

### 4.3 FCF vs TS3 — Bet3 替換方案

**假說**: 將 3注策略的 Bet3 從 TailBalance (尾數均衡) 替換為 FreqOrt (頻率正交) 可提升 Edge。此即 FCF (Fourier+Cold+FreqOrt) vs TS3 (Fourier+Cold+TailBalance)。

**原始宣稱** (來自 C(5,3)=10 組搜尋):
- FCF Edge: +1.58%
- TS3 Edge: +1.05%

**1v1 配對驗證結果** (1500期):

| 策略 | M3+ 命中 | M3+ 率 | Edge |
|------|---------|--------|------|
| TS3 | 98/1500 | 6.53% | +19.29% |
| FCF | 95/1500 | 6.33% | +15.64% |
| 差值 | TS3多3期 | -0.20% | -3.65% |

**統計檢定**:
- McNemar 配對: b=11(FCF獨贏), c=14(TS3獨贏), χ²=0.36, p(單側)=0.726
- Permutation Test: real Δ=-0.20%, p=0.767
- Cohen's d: -0.59 (FCF 方向錯誤)
- 三窗口: TS3 在 150/500/1500 期全部領先

**Bet3 單獨救援分析**:
- Bet1+Bet2 皆未中的期數: 1429 期
- TailBalance 獨力救援: 27 期 (1.89%)
- FreqOrt 獨力救援: 24 期 (1.68%)
- 差值: TS3 多救 3 期

**失敗原因**:
1. 原始結論來自 C(5,3)=10 路多重搜尋，Bonferroni 未通過
2. 1v1 直接對比完全消除多重比較問題後，FCF 甚至落後
3. TailBalance 的尾數分散覆蓋在實測中優於純頻率排序

**最終裁決**: **維持 TS3 (MAINTAIN_TS3)**

### 4.4 覆蓋率最大化與信號模型

**問題摘要**: 多份外部研究報告聲稱發現新的預測信號，但均存在基線計算錯誤：

| 報告 | 宣稱2注基準 | 正確2注基準 | 偏差 |
|------|-----------|-----------|------|
| 報告A (覆蓋策略) | 3.27% | 3.69% | -11.3% (低估) |
| 報告B (涵蓋率信號) | 4.73% | 3.69% | +28.3% (高估, z=2.15) |

**具體問題**:
- 報告 A: "AI_TS3_Mock" 不是真正的 TS3 策略實現
- 報告 B: 2注隨機基準 4.73% 偏高 28.3%，屬於統計顯著的系統性偏差 (z=2.15)
- 兩份報告的策略定義與本系統不一致（單信號 × 所有注數 ≠ 正交組合策略）
- 報告中提供的預測號碼來自未通過驗證的策略

**最終裁決**: **不予採納，數據不可靠**

## 六、統計學討論

### 6.1 Edge 的物理意義

即使是最佳的 5注策略 (Edge +1.77%)，其實際含義是：

- **每 100 期期望多中 1.77 注** (相對隨機)
- **每期花費**: 5注 × 50元 = 250元
- **中 M3 (3星) 獎金**: 約 1,000~2,000 元 (大樂透 M3 約為普獎)
- **期望值**: 仍為負 (因中獎獎金遠不及購買成本的統計累積)

結論：統計意義上的 Edge 不等於經濟意義上的正期望值。

### 6.2 為何 Edge 存在但很小？

可能的物理機制：
1. **機械偏差**: 開球機器的微小不均勻性可能造成某些號碼略高的出現概率
2. **更換球組**: 定期更換球組可能造成短暫的頻率偏移
3. **時序慣性**: 連續開獎之間可能存在微弱的自相關性

這些效應的量級（~1-2%相對Edge）與我們的觀測結果一致。

### 6.3 P3 Permutation Test 的意義

洗牌後 Edge 降至 0.22% (BL) / 0.31% (PL)，而真實 Edge 為 1.77% / 2.30%：

- **信號佔比**: (1.77-0.22)/1.77 = 88% 的 Edge 來自時序結構
- **分布性質佔比**: 0.22/1.77 = 12% 來自號碼分布性質（選號偏好本身的微弱優勢）
- **Cohen's d > 2.0**: 效果量極大，信號非偶然

但需注意：200次洗牌的 p=0.030 意味著在多次獨立實驗中，約 3% 的可能性是假陽性。這是我們設定的風險容忍度。

### 6.4 多重比較的教訓

本研究反覆遇到「多重比較陷阱」：

| 場景 | 比較次數 | Bonferroni α' | 結果 |
|------|---------|--------------|------|
| RGF (3 State × 6 Formula) | 18 | 0.0028 | 全部失敗 |
| C(5,3)=10 組合搜尋 | 10 | 0.005 | FCF p=0.033 > 0.005 |
| FCF vs TS3 (1v1) | 1 | 0.05 | p=0.726 >>> 0.05 |
| 5注策略 (單一驗證) | 1 | 0.05 | p=0.008 < 0.05 |

規律：**搜尋越廣泛，越容易找到虛假信號。正確的做法是先假設再驗證，而非從數據中挖掘假設。**

## 七、結論與建議

### 7.1 核心結論

1. **大樂透與威力彩存在微弱但統計可測的時序信號**，主要體現在 FFT 週期結構、冷熱號交替、Markov 轉移等低階特徵中。

2. **高階特徵（Regime分類、乘法交互、覆蓋率優化）均未能提供額外的統計顯著邊緣**。這暗示可利用信號的資訊量已被現有低階特徵充分捕捉。

3. **現行策略已接近可利用信號的上限**：
   - 大樂透: Edge +1.77% (z=2.40)
   - 威力彩: Edge +2.30% (p=0.015)
   - 進一步提升空間極為有限

4. **經濟層面不具備正期望值**：統計 Edge 不足以覆蓋購買成本。彩券投注應視為娛樂消費而非投資行為。

### 7.2 操作建議

| 建議 | 說明 |
|------|------|
| 維持現行策略 | TS3+M+FO (BL 5注) 與 PP3 (PL 3注) 不做變更 |
| 拒絕未驗證的「新發現」 | 任何新策略需通過完整三階段驗證才能採納 |
| 定期監控 Edge 衰減 | 每季度重跑 150/500/1500 三窗口驗證 |
| 注碼管理 | 固定注數，不因短期波動調整 |
| 理性預期 | Edge 不等於盈利，仍為負期望值遊戲 |

### 7.3 未來研究方向

在 DSE 路線圖中，以下方法尚未測試，但基於 RGF 的失敗經驗，預期突破概率極低：

- **Symbolic Regression (GP)**: 自動搜尋公式空間。風險：與 RGF 相同的多重比較問題。
- **Hurst Exponent**: 判斷慣性 vs 反轉。風險：特徵可能與 FFT 高度共線。
- **Graph Embedding**: 捕捉非線性共現。風險：計算成本高，信號可能在噪音中。

**建議策略**: 將研究重心從「新特徵開發」轉向「現有策略的穩定性監控與衰減偵測」。信號有限的系統中，防禦性維護比攻擊性搜尋更有價值。
