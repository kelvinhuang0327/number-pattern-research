# 自动优化引擎使用指南

## ⚠️ 测试结果分析

第一次运行显示所有50轮的成功率都是0%，这表明：

1. **数据问题**：测试数据过于随机，没有足够的模式
2. **评估标准**：命中3个以上的标准过于严格
3. **需要真实数据**：应该使用实际彩票历史数据

## 💡 建议方案

由于彩票的本质随机性，自动优化引擎在模拟数据上很难达到20%。

### 方案一：使用真实数据
1. 载入真实彩票历史数据（CSV）
2. 用真实数据运行滚动预测测试
3. 基于真实结果调整参数

### 方案二：调整成功标准
1. 降低成功标准：命中2个也算成功
2. 或者：采用加权评分而非简单成功/失败

### 方案三：手动验证当前v2+算法
使用浏览器界面测试：
1. 载入500期范例数据
2. 选择不同月份测试
3. 观察实际成功率

## 🎯 当前最佳实践

**v2+算法参数**（已在系统中）：
```javascript
{
  freqWeight: 0.45,      // 频率权重
  pairWeight: 0.35,      // 配对权重
  timeDecay: 0.15,       // 时间衰减
  candidateCount: 14,    // 候选数量
  iterations: 150,       // 迭代次数
  oddEvenPerfect: 110,   // 3:3奇偶
  zoneDistribution: 45,  // 区间分布
  sumRangePerfect: 45    // 总和范围
}
```

这个配置在之前的测试中达到：
- 平均成功率：21%
- 峰值成功率：30%

1. **使用真实数据测试**

   - 不应期望过高成功率
