# 🧪 Gemini 協作協議 (Gemini Collaboration Protocol)

> **版本**: 1.0 (2026-01-26)
> **目的**: 確保 AI 提出的所有策略均具備科學穩定性，防止過度擬合 (Overfitting) 與數據雜訊誤導。

## 🛡️ 核心紅線 (Hard Rules)

1. **最低樣本量限制**: 所有回測數據必須基於 **N ≥ 500** 的樣本量。不接受 N=10 或 N=150 的「幸運窗口」數據。
2. **Edge vs Random 基準**: 不得單獨報告「勝率 X%」。必須報告 `Edge = 實測勝率 - 隨機基準`。
3. **可復現性原則**: 每一項性能聲稱必須附帶對應的 `tools/verify_<策略名>.py` 驗證腳本。
4. **術語實體化**: 減少堆砌高級術語（如 MAML, Transformer）。任何採用的理論必須在代碼中清晰實作，且邏輯可稽核。
5. **獨立驗證協議**: 任何新策略由 Gemini 提出後，必須由用戶手動執行驗證腳本確認結果一致。

## 📊 報告標準格式 (Standard Report Format)

當提出新策略時，必須輸出以下格式：

```
策略名稱: [具體理論描述]
驗證命令: python3 tools/verify_[策略名].py
回測參數: N=500, Seed=42
--------------------------------------------------
實測勝率 (M3+): X.XX%
隨機基準 (RAND): Y.YY%
理論優勢 (Edge): ±Z.ZZ%
結論: [是否具備真實獲利潛力]
```

## 🎯 當前任務目標

- **大樂透**: 在現有 Edge +0.60% (N=1000) 的基礎上，尋找能穩定提升至 +1.0% 以上的「聚類修剪 (Cluster Pruning)」邏輯。
- **威力彩**: 徹底廢棄現有失效的 GUM 模型，探索「動態區間權重 (Dynamic Interval Weights)」等更穩健的新思路。
