## Overview

This document defines the **normalized row contract** that the V2 ARTIFACT_ONLY parser must produce. Every row generated by the parser must conform to this schema exactly.

**Key Principle**: One row per historical draw × strategy combination. Rows are **ARTIFACT_RECONSTRUCTED_RETROSPECTIVE** (generated from artifact sources, not from live predictions).

---

## Row Contract (TypeScript-style)

```typescript
interface ArtifactReconstructedRow {
  // === Identity & Lottery Info ===
  strategy_id: string;           // e.g., "biglotto_ts3_acb_4bet"
  lottery_type: string;          // BIG_LOTTO | POWER_LOTTO | DAILY_539
  
  // === Draw Identifiers ===
  target_draw: string;           // e.g., "115000001" (TEXT, zero-padded)
  target_date: string;           // e.g., "2026/01/02" (YYYY/MM/DD format)
  
  // === Predictions & Actuals ===
  predicted_numbers: number[];   // Sorted array, 6 numbers for BIG/POWER, 5 for 539
  predicted_special: number | null;  // POWER_LOTTO only, null for others
  actual_numbers: number[];      // Sorted array, 6 numbers for BIG/POWER, 5 for 539
  actual_special: number | null; // POWER_LOTTO only, null for others
  
  // === Hit Analysis ===
  hit_numbers: number[];         // Intersection of predicted & actual (sorted)
  hit_count: number;             // len(hit_numbers)
  special_hit: number;           // 1 if predicted_special == actual_special && special != null, else 0
  
  // === Truth & Provenance ===
  truth_level: string;           // ARTIFACT_RECONSTRUCTED_RETROSPECTIVE (constant)
  source: string;                // v2_artifact_only_parser_dryrun (constant)
  provenance_source: string;     // e.g., "rejected/ts3_acb_4bet_biglotto.json"
  provenance_hash: string;       // SHA256(artifact_json_bytes + row_method)
  
  // === Dry-run Safeguards ===
  dry_run_only: boolean;         // Always true (1 in SQLite)
  history_window_end: string;    // Last draw used in training window (must be < target_draw)
  leakage_guard_pass: boolean;   // true if history_window_end < target_draw
  controlled_apply_id: null;     // Always null for dry-run (no DB control ID yet)
}
```

## Field Requirements

### Identity & Lottery Info

| Field | Type | Format | Example | Validation |
|-------|------|--------|---------|-----------|
| `strategy_id` | TEXT | kebab-case | biglotto_ts3_acb_4bet | Must match registry (lines 374-400 in replay_strategy_registry.py) |
| `lottery_type` | TEXT | ENUM | BIG_LOTTO | Only ARTIFACT_ONLY lotteries: BIG_LOTTO, POWER_LOTTO, DAILY_539 |

### Draw Identifiers

| Field | Type | Format | Example | Validation |
|-------|------|--------|---------|-----------|
| `target_draw` | TEXT | 9-char zero-padded | 115000001 | Must exist in lottery_history (all draws) or actual draw data |
| `target_date` | TEXT | YYYY/MM/DD | 2026/01/02 | Must match draw_date from lottery history |

**Draw Lookup Rule**: 
- For BIG_LOTTO / POWER_LOTTO: Query lottery history or actual draw table
- For DAILY_539: Query daily539_draws or 539 history table
- target_draw and target_date must be consistent (same draw)

### Predictions & Actuals

| Field | Type | Format | Example | Validation |
|-------|------|--------|---------|-----------|
| `predicted_numbers` | JSON array | Sorted integers | [1, 5, 10, 15, 20, 25] | 6 numbers for BIG/POWER; 5 for 539; must be sorted ascending |
| `predicted_special` | INTEGER or NULL | 1-49 or null | 42 (POWER_LOTTO), null (others) | Only POWER_LOTTO: 1-49. Null for BIG_LOTTO and DAILY_539 |
| `actual_numbers` | JSON array | Sorted integers | [3, 7, 11, 16, 22, 28] | 6 numbers for BIG/POWER; 5 for 539; must be sorted ascending |
| `actual_special` | INTEGER or NULL | 1-49 or null | 42 (POWER_LOTTO), null (others) | Only POWER_LOTTO: 1-49. Null for BIG_LOTTO and DAILY_539 |

**Lottery-specific Constraints**:
- **BIG_LOTTO**: 6 regular + null special (predicted_special, actual_special both null)
- **POWER_LOTTO**: 6 regular + 1 special (predicted_special, actual_special both 1-49 or both null)
- **DAILY_539**: 5 regular + null special (predicted_special, actual_special both null)

### Hit Analysis

| Field | Type | Computation | Validation |
|-------|------|-------------|-----------|
| `hit_numbers` | JSON array | sorted(set(predicted_numbers) ∩ set(actual_numbers)) | 0-6 elements; must be subset of both |
| `hit_count` | INTEGER | len(hit_numbers) | 0-6 for BIG/POWER; 0-5 for 539 |
| `special_hit` | INTEGER | 1 if (predicted_special == actual_special and special != null), else 0 | 0 or 1 only |

**Example Computation**:
```python
predicted = [1, 5, 10, 15, 20, 25]
actual = [3, 7, 11, 16, 22, 28]
hit_numbers = sorted(set(predicted) & set(actual))  # []
hit_count = len(hit_numbers)  # 0
special_hit = 0

predicted = [1, 5, 10, 15, 20, 25]
actual = [1, 3, 7, 11, 16, 22]
hit_numbers = [1]
hit_count = 1
special_hit = 0 (or computed if POWER_LOTTO)
```

### Truth & Provenance

| Field | Type | Value | Purpose |
|-------|------|-------|---------|
| `truth_level` | TEXT | ARTIFACT_RECONSTRUCTED_RETROSPECTIVE | Fixed constant for all rows |
| `source` | TEXT | v2_artifact_only_parser_dryrun | Fixed constant (parser name) |
| `provenance_source` | TEXT | rejected/strategy_name.json | Artifact filename this row came from |
| `provenance_hash` | TEXT | SHA256 hex digest | Reproducibility marker (see below) |

**Provenance Hash Computation**:
```python
import hashlib
import json

artifact_json_str = read_artifact_file(provenance_source)
row_method = "artifact_reconstructed"
combined = artifact_json_str + row_method
provenance_hash = hashlib.sha256(combined.encode()).hexdigest()
```

This ensures every row can be traced back to:
1. The exact artifact source file it came from
2. The method used to reconstruct it (deterministic)

### Dry-run Safeguards

| Field | Type | Value | Purpose |
|-------|------|-------|---------|
| `dry_run_only` | INTEGER | 1 (true) | Marks all rows as dry-run candidates, not production |
| `history_window_end` | TEXT | 9-char draw ID | Last draw in strategy's training window (must be < target_draw) |
| `leakage_guard_pass` | INTEGER | 1 (true) | Passes leakage guard iff CAST(history_window_end AS INTEGER) < CAST(target_draw AS INTEGER) |
| `controlled_apply_id` | TEXT | NULL | Always NULL for dry-run (reserved for controlled apply phase) |

**Leakage Guard Logic**:
```python
history_end_int = int(history_window_end)
target_int = int(target_draw)
leakage_guard_pass = (history_end_int < target_int)
```

If leakage_guard_pass is false, the row **must be rejected** before output.

## Validation Gates (Parser Must Enforce)

### Gate 1: Leakage Protection
- **Rule**: `CAST(history_window_end AS INTEGER) < CAST(target_draw AS INTEGER)`
- **Action**: Reject row if false
- **Purpose**: Prevent future data leakage into training window

### Gate 2: Completeness
- **Rule**: Every field except `special_hit` must be non-null and valid
- **Action**: Reject row if any null found
- **Purpose**: Ensure full row fidelity

### Gate 3: Format Validation
- **Rule**: 
  - Numbers in sorted ascending order
  - hit_count = len(hit_numbers)
  - special_hit ∈ {0, 1}
  - draw IDs are 9-digit numeric strings
  - dates match YYYY/MM/DD format
- **Action**: Reject row if any violation
- **Purpose**: Detect parsing errors

### Gate 4: Lottery-specific Constraints
- **Rule**:
  - BIG_LOTTO: predicted_special and actual_special must be null
  - POWER_LOTTO: predicted_special and actual_special must both be null or both 1-49
  - DAILY_539: predicted_special and actual_special must be null
- **Action**: Reject row if violated
- **Purpose**: Ensure lottery correctness

### Gate 5: Determinism
- **Rule**: Running parser twice with same input must produce identical output (bit-for-bit for provenance_hash)
- **Action**: Verify during testing
- **Purpose**: Ensure reproducibility and auditability

### Gate 6: Provenance Traceability
- **Rule**: Every row must have valid provenance_source (file exists) and provenance_hash (consistent)
- **Action**: Log provenance_source in summary
- **Purpose**: Enable row reconstruction audit

## Parser Implementation Requirements

### CLI Interface
```bash
python3 scripts/v2_artifact_only_parser_dryrun.py \
  [--strategy-id <id> | --all] \
  [--output-jsonl <path>] \
  [--output-summary <path>] \
  [--dry-run]  # default true, creates no DB records
```

### Exit Codes
- `0`: All rows generated, all validation gates passed
- `1`: Some rows rejected due to validation gates
- `2`: Fatal error (artifact file not found, parse error)

### Logging
- Every rejected row logged with reason
- Provenance_hash consistency verified
- Leakage guard results per strategy

## Determinism Guarantees

The parser guarantees:
1. **Input Determinism**: Same artifact file + same draw history → same output rows
2. **Output Determinism**: Same strategy_id + --all flag → same JSONL file (byte-for-byte identical)
3. **Hash Determinism**: provenance_hash computed from (artifact JSON + method string) → reproducible
4. **No Randomness**: Zero random number generation; all selection based on deterministic draw dates
