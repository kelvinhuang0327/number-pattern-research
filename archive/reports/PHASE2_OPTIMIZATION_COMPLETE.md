### 1. ✅ 多維度偏差分析（高優先）

**文件**: `lottery-api/models/unified_predictor.py`
**方法**: `deviation_predict()`
**行數**: 241-407

**實施內容**:

從單一頻率偏差擴展到5個維度的綜合分析：

#### 維度1: 頻率偏差（權重30%）- 原有邏輯
- 計算每個號碼的出現頻率
- 使用Z-score評估偏差程度
- 識別過熱和過冷的號碼

#### 維度2: 區域偏差（權重25%）- 新增
- 將49個號碼分為5個區域（1-10, 11-20, 21-30, 31-40, 41-49）
- 計算每個區域的期望值和實際值
- 補償出現不足的區域

#### 維度3: 奇偶偏差（權重20%）- 新增
- 統計奇數和偶數的出現次數
- 理論上奇偶應各佔50%
- 補償比例失衡的一方

#### 維度4: 大小偏差（權重15%）- 新增
- 小號（1-24）vs 大號（25-49）
- 計算大小號的平衡狀態
- 補償比例失衡的一方

#### 維度5: 遺漏值偏差（權重10%）- 新增
- 計算每個號碼距離上次出現的期數
- 給予長期未出現的號碼更高分數
- 避免忽略冷門號碼

**核心代碼**:
```python
# 初始化綜合評分
scores = np.zeros(max_num + 1)

# 維度1: 頻率偏差（30%）
freq_scores = calculate_frequency_deviation(history)
scores += freq_scores * 0.3

# 維度2: 區域偏差（25%）
zone_scores = calculate_zone_deviation(history)
scores += zone_scores * 0.25

# 維度3: 奇偶偏差（20%）
odd_even_scores = calculate_odd_even_deviation(history)
scores += odd_even_scores * 0.2

# 維度4: 大小偏差（15%）
high_low_scores = calculate_high_low_deviation(history)
scores += high_low_scores * 0.15

# 維度5: 遺漏值偏差（10%）
gap_scores = calculate_gap_deviation(history)
scores += gap_scores * 0.1
```

**測試結果**（50期回測）:
- 勝率: 0.00%
- 平均匹配: 0.52
- 匹配分布: 0中=26, 1中=22, 2中=2

**分析**:
- 多維度分析增加了複雜度，但短期測試結果不理想
- 可能需要調整各維度的權重
- 建議擴大測試樣本（100-200期）以獲得更可靠結果

### 2. ✅ 元學習動態權重調整器（高優先）

**文件**: `lottery-api/models/dynamic_weight_adjuster.py`
**類**: `DynamicWeightAdjuster`

**實施內容**:

創建了一個完整的動態權重調整系統：

#### 核心功能

1. **性能追蹤**
   - 記錄每個方法的最近20次預測結果
   - 使用雙端隊列自動限制歷史大小
   - 支持持久化（保存/載入狀態）

2. **動態權重計算**
   - **平均表現**（60%）: 匹配越多越好
   - **穩定性**（20%）: 標準差越小越好
   - **趨勢**（20%）: 近期上升趨勢加分

3. **自動調整**
   - 表現好的方法自動提升權重（最高40%）
   - 表現差的方法自動降低權重（最低5%）
   - 確保所有權重總和為100%

#### 使用示例

```python
from models.dynamic_weight_adjuster import DynamicWeightAdjuster

# 創建調整器
adjuster = DynamicWeightAdjuster(
    history_size=20,
    persistence_file='data/weights.json'
)

# 更新表現
adjuster.update_performance(
    method='entropy',
    actual_numbers=[1, 2, 3, 4, 5, 6],
    predicted_numbers=[1, 2, 7, 8, 9, 10]  # 2個匹配
)

# 獲取當前權重
weights = adjuster.get_weights()
# {'entropy': 0.298, 'deviation': 0.183, ...}

# 獲取表現摘要
summary = adjuster.get_performance_summary()
```

**測試結果**:
- ✅ 表現好的方法（entropy，平均2.5個匹配）權重提升到 29.8%
- ✅ 表現差的方法（deviation，平均0.5個匹配）權重降低到 18.3%
- ✅ 自動正規化確保總和為 100%

**優勢**:
- 完全自動化，無需手動調整
- 考慮表現、穩定性和趨勢三個維度
- 支持持久化，可跨會話保留狀態

### 3. ✅ 自適應時間窗口計算器（中優先）

**文件**: `lottery-api/models/adaptive_window.py`
**類**: `AdaptiveWindowCalculator`

**實施內容**:

根據號碼的波動性自動調整時間窗口大小：

#### 核心邏輯

1. **收集出現位置**
   - 記錄號碼在歷史中所有出現的位置
   - 計算相鄰出現的間隔

2. **波動性分析**
   - 計算間隔的平均值和標準差
   - 使用變異係數（CV = 標準差 / 平均值）衡量波動性

3. **窗口決策**
   - **高波動（CV > 1.0）**: 小窗口（20期）- 只看最近
   - **中波動（CV > 0.5）**: 中窗口（50期）- 平衡考慮
   - **低波動（CV ≤ 0.5）**: 大窗口（100期）- 看更長歷史

#### 使用示例

```python
from models.adaptive_window import AdaptiveWindowCalculator

# 創建計算器
calculator = AdaptiveWindowCalculator(
    min_window=20,
    medium_window=50,
    max_window=100
)

# 計算單個號碼的最佳窗口
window = calculator.calculate_optimal_window(
    number=13,
    history=history_data
)
# 返回: 20, 50 或 100

# 計算所有號碼的窗口
all_windows = calculator.calculate_all_windows(
    history=history_data,
    min_num=1,
    max_num=49
)

# 獲取統計信息
stats = calculator.get_window_statistics(all_windows)
```

**測試結果**（模擬數據）:
- 平均窗口: 47.6 期
- 小窗口（20期）: 9個號碼（18%）
- 中窗口（50期）: 37個號碼（76%）
- 大窗口（100期）: 3個號碼（6%）

**優勢**:
- 每個號碼使用最適合的窗口大小
- 高波動號碼反應更快
- 低波動號碼參考更長歷史
