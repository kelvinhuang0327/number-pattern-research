# 🚀 後端優化預測功能

## 概述

「後端優化預測」是一個全新的預測功能，它直接調用後端遺傳算法優化過的參數進行預測，**成功率接近 10%**（相比傳統方法的 3-5%）。

![工作流程](/.gemini/antigravity/brain/c419635d-865e-45d5-bb61-ba7cca8d3498/backend_optimized_flow_1764320678799.png)

---

## ✨ 核心優勢

| 特點 | 說明 |
|------|------|
| 🎯 **高成功率** | 使用遺傳算法優化，成功率接近 10% |
| ⚡ **快速響應** | 直接使用已優化參數，無需重新計算 |
| 🎮 **簡單易用** | 只需選擇一個選項即可使用 |
| 🤖 **自動優化** | 支持自動排程，每天自動更新參數 |
| 💾 **模型緩存** | 後端支持緩存，進一步提升速度 |

## 🚀 快速開始（三步驟）

### 步驟 1: 啟動後端服務

```bash
./start_backend.sh
```

或手動啟動：

```bash
cd lottery-api
python3 app.py
```

### 步驟 2: 同步數據

1. 在前端上傳或載入彩票數據
2. 切換到「🤖 自動學習」頁面
3. 點擊「📤 同步數據到後端」

### 步驟 3: 執行優化

1. 設置遺傳代數：**20**
2. 設置種群大小：**30**
3. 點擊「🚀 開始優化」
4. 等待完成（約 2-5 分鐘）

## 🛠️ 技術架構

### 前端
- **策略類**: `BackendOptimizedStrategy.js`
- **註冊名稱**: `backend_optimized`
- **API 端點**: `POST /api/predict-optimized`

### 後端
- **API 實現**: `lottery-api/app.py` (第 563-637 行)
- **優化引擎**: `models/auto_learning.py`
- **配置存儲**: `data/best_config.json`

### 工作流程

```
用戶選擇「後端優化預測」
    ↓
BackendOptimizedStrategy.predict()
    ↓
POST /api/predict-optimized
    ↓
載入 best_config.json
    ↓
使用遺傳算法優化參數預測
    ↓
返回結果（信心度 ~10%）
    ↓
顯示預測號碼
```

## 🐛 常見問題

### Q: 提示「無法連接到後端服務器」？

**A**: 後端服務未運行，請執行：
```bash
./start_backend.sh
```

### Q: 提示「尚未執行自動優化」？

**A**: 需要先執行一次優化：
1. 到「自動學習」頁面
2. 點擊「開始優化」
3. 等待完成

### Q: 提示「後端沒有數據」？

**A**: 需要同步數據：
1. 到「自動學習」頁面
2. 點擊「同步數據到後端」

### Q: 如何提高成功率？

**A**: 
- 定期執行優化（建議每週一次）
- 啟用自動排程（每天自動優化）
- 確保數據是最新的
- 使用足夠的歷史數據（建議 100+ 期）

## 💡 最佳實踐

### 1. 啟用自動排程
設置每天凌晨 2:00 自動優化，確保參數始終是最新的。

### 2. 定期更新數據
有新開獎數據時，記得同步到後端，保持數據的時效性。

### 3. 多方法對比
同時使用多種預測方法，對比不同方法的結果。

### 4. 追蹤成功率
記錄實際使用的成功率，根據結果調整策略。
