## Leading-Zero Handling

⚠️ Critical: 4-digit draws have more leading-zero cases.

| Example result | DB storage | Correct | Wrong |
|---------------|------------|---------|-------|
| `0012` | `[0, 0, 1, 2]` | ✅ | `[12]` or `12` |
| `0100` | `[0, 1, 0, 0]` | ✅ | `[100]` or `100` |
| `5678` | `[5, 6, 7, 8]` | ✅ | — |

**Ingestion requirement**: Must store as `json.dumps([d1, d2, d3, d4])` where each `d_i` is a
single int 0-9. Do NOT collapse to a single integer. Do NOT use string representation.

**Verification step after ingestion**: Run equivalent of `special3_leading_zero_check` for
4_STAR — verify all rows have `json.loads(numbers)` giving a list of exactly 4 ints in [0-9].
